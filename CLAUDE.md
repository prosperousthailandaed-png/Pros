# บันทึกการแก้ไข (Changelog)

## 2026-07-01 (รอบ 2) — เอา Eyebrow ออกจากไทม์ไลน์ + ทำ Wordmark "PROSPEROUS" แนวตั้ง

### สิ่งที่ขอ
1. ตรงท้ายส่วน Core Values ให้สะกดเต็ม "P R O S P E R O U S" (ครบ 10 ตัว
   ตามชื่อบริษัทจริง แทนที่จะสะกดแค่ "PROSPER" 7 ตัวเหมือนเดิม) และเรียง
   **ลงมาแนวตั้ง** แทนแถวนอน
2. ในส่วนไทม์ไลน์ ไม่ต้องมีป้าย eyebrow (ข้อความตัวเล็กพิมพ์ใหญ่สีแดง
   เช่น "จุดเริ่มต้น", "จุดเปลี่ยน" ที่อยู่เหนือหัวข้อแต่ละจุด)
3. ให้ดีไซน์โดยรวมของหน้า About ดู Professional ขึ้น

### สมมติฐานที่ตั้งไว้ (สำคัญ — ถ้าไม่ตรงใจ บอกให้แก้ต่อได้เลย)
คุณค่าหลัก 7 ข้อ (Professionalism, Rescue Excellence, Outreach for Society,
Safety First, Passion for Life, Education & Empowerment, Responsibility &
Reliability) มีอยู่แล้วแค่ 7 ข้อ ตรงกับตัวอักษร 7 ตัวแรกของคำว่า "PROSPEROUS"
เท่านั้น ส่วน O, U, S ตัวท้าย **ยังไม่มีคำนิยามคุณค่าใด ๆ ในเนื้อหาต้นฉบับ**
ผมจึงตีความว่าที่ขอมาคือ "wordmark" หรือลายเซ็นแบรนด์ที่สะกดชื่อบริษัทเต็ม ๆ
เพื่อความสวยงาม/เป็นมืออาชีพ ไม่ใช่การเพิ่มคุณค่าใหม่อีก 3 ข้อ — ถ้าจริง ๆ
อยากได้คำนิยามคุณค่าเพิ่มสำหรับ O, U, S (การ์ดคุณค่าเพิ่มอีก 3 ใบ) ต้องบอก
ว่าแต่ละตัวควรสื่อความหมายว่าอะไร แล้วจะเพิ่มการ์ดให้ครบ 10 ใบในรอบถัดไป

### สิ่งที่แก้ไป

1. **Core Values section (`app/about/page.tsx` + `app/globals.css`)**
   - เปลี่ยนจาก `.prosper-word` (กล่องแดงสี่เหลี่ยมเรียงแถวนอน สะกดแค่ PROSPER)
     เป็น `.prosper-mark` — ตัวอักษรขนาดใหญ่สไตล์ typographic เรียงลงมา
     แนวตั้ง สะกดครบ "P R O S P E R O U S" ทั้ง 10 ตัว มีเส้นบาง ๆ คั่นแต่ละตัว
     ตัวอักษร 7 ตัวแรก (ที่มีคุณค่ากำกับ) เน้นด้วยสีแดงแบรนด์ ส่วน O, U, S
     ท้ายคำเป็นสีเทาเข้มธรรมดา แยกให้เห็นชัดว่าอันไหนมีความหมายกำกับอยู่
   - จัดหัวข้อส่วนนี้ใหม่เป็น 2 คอลัมน์บนจอกว้าง (`.values-head`): ซ้ายเป็น
     eyebrow + หัวข้อ + คำอธิบายสั้น ๆ, ขวาเป็น wordmark แนวตั้ง — ดูเป็นงาน
     ออกแบบเชิงบรรณาธิการ (editorial) มากกว่าเดิมที่เป็นกล่องสีแดงเรียงแถว
   - หัวข้อเปลี่ยนจาก "P · R · O · S · P · E · R" เป็น "คุณค่าที่สะกดจากชื่อของเรา"
     พร้อมคำอธิบายสั้น ๆ อธิบายที่มาของ 7 ตัวอักษรแรก กันสับสนว่าทำไมมี 10
     ตัวอักษรแต่การ์ดคุณค่าด้านล่างมีแค่ 7 ใบ

2. **ไทม์ไลน์ (`app/about/page.tsx` + `app/globals.css`)**
   - เอา `<span className="timeline-year">...</span>` (ป้าย eyebrow: "จุดเริ่มต้น",
     "จุดเปลี่ยน", "ก้าวแรก", "ความเชื่อที่ยึดมั่น") ออกทั้งหมดตามที่ขอ
   - เปลี่ยนจุดวงกลมบนเส้นไทม์ไลน์ (`.timeline-dot`) จากวงกลมเปล่า ให้มี
     **เลขลำดับ (1–4)** อยู่ข้างในแทน — ยังบอกลำดับเรื่องราวได้แต่ไม่ใช่ eyebrow
     (ดีไซน์เดียวกับวงกลมเลข `.m-num` ที่ใช้ในส่วนพันธกิจอยู่แล้ว ให้ภาษาภาพ
     ตรงกันทั้งหน้า)
   - ห่อเนื้อหาแต่ละจุด (หัวข้อ, พารากราฟ, คำพูดเน้น) ด้วยการ์ดสีขาว
     (`.timeline-card`) มีเงานุ่ม ๆ และขอบมน แทนการปล่อยข้อความลอยอยู่บนพื้น
     `.mist` เฉย ๆ — ทำให้แต่ละจุดดูเป็นชิ้นเป็นอันและมีระดับความเป็นมืออาชีพ
     มากขึ้น

## 2026-07-01 — แก้ Responsive ทั้งเว็บ + ทำ Effect ให้สมูท + ออกแบบหน้า About ใหม่

### ปัญหาที่เจอตอนเริ่มงาน (สำคัญ อ่านก่อนแก้ต่อ)
ไฟล์ `app/globals.css` ที่ได้รับมามีแค่การตั้งค่า Tailwind `@theme` (สี/ฟอนต์)
แต่ **ไม่มี CSS จริงของ class ที่ใช้ทั่วเว็บเลย** เช่น `.hero`, `.wrap`, `.narrow`,
`.pgrid`, `.pcard`, `.mist`, `.reveal` ฯลฯ และไม่มีตัวแปรที่ `components/Navbar.tsx`
เรียกใช้ตรง ๆ อย่าง `--black`, `--red`, `--ink`, `--muted-d`, `--txt-d`, `--line-d`
เลยด้วย พูดง่าย ๆ คือไฟล์ที่อยู่ในโปรเจกต์ตอนนั้น "คนละเวอร์ชัน" กับเว็บที่ขึ้นจริงบน
Vercel (ที่ผู้ใช้ส่งภาพหน้าจอมาให้ดู)

**ทางแก้ที่เลือก (ตามที่ผู้ใช้ยืนยัน):** เขียน `app/globals.css` ขึ้นใหม่ทั้งหมด
โดยอ้างอิงหน้าตาจากภาพหน้าจอ + ชื่อ class ที่มีอยู่แล้วใน JSX เป็นหลัก
**หน้าตาอาจไม่ตรง 100% กับต้นฉบับที่ deploy ไว้จริง** — ถ้าจะให้ตรงเป๊ะ ต้องขอ
ไฟล์ `globals.css` ตัวจริงจาก production repo มาเทียบอีกที

### สิ่งที่แก้ไป

1. **`app/globals.css` — เขียนใหม่ทั้งไฟล์**
   - เพิ่มตัวแปรสี/ระยะที่ Navbar และหน้าอื่น ๆ เรียกใช้แต่ไม่เคยถูกประกาศ
     (`--black`, `--red`, `--red-dark`, `--ink`, `--line`, `--line-d`, `--txt-d`,
     `--muted-d`, `--ease`, `--speed`)
   - เขียน class ที่ขาดไปทั้งหมดของหน้าแรก/หน้า about/Footer แบบ responsive
     โดยใช้ `clamp()` กับ font-size/spacing แทนการ set ค่าตายตัวทีละ breakpoint
     ทำให้ขนาดข้อความ/ระยะห่างไหลลื่นตามขนาดจอจริง ไม่กระโดดเป็นขั้น ๆ
   - การ์ด (`.pgrid/.cgrid/.ngrid`) ใช้ `grid-template-columns: repeat(auto-fit, minmax(260px, 1fr))`
     แทนการ fix จำนวนคอลัมน์ตาม breakpoint ตายตัว จึงพอดีกับทุกขนาดจอ/ทุก platform
     โดยอัตโนมัติ
   - เพิ่ม transition ที่ใช้ตัวแปรกลาง `--ease` / `--speed` ให้ hover การ์ด, ลิงก์
     "ดูเพิ่มเติม", ปุ่ม ฯลฯ ลื่นไหลไปในทิศทางเดียวกันทั้งเว็บ
   - เพิ่ม `@media (prefers-reduced-motion: reduce)` ปิดแอนิเมชันทั้งหมด
     (reveal + เส้น ECG) ให้ผู้ใช้ที่ตั้งค่าระบบไว้แบบนั้น — ตามหลัก accessibility
   - แก้ font-family: เดิมอ้างชื่อฟอนต์ตรง ๆ (`"Anuphan"`) ทำให้ฟอนต์ที่โหลดผ่าน
     `next/font/google` ใน `layout.tsx` (ตัวแปร `--font-anuphan` / `--font-ibm`)
     ไม่เคยถูกใช้งานจริงเลย แก้เป็นอ้างตัวแปรนั้นก่อน แล้วค่อย fallback เป็นชื่อฟอนต์ตรง ๆ

2. **หน้าแรก (`app/page.tsx`)**
   - ไม่ได้แก้โครงสร้าง JSX (เนื้อหาเดิมทั้งหมด) แต่ให้ `.hero` เป็น `min-height: 100dvh`
     (เต็มจอแบบเว็บแบรนด์ใหญ่ ตามที่ผู้ใช้เลือก) วิดีโอ cover เต็มพื้นที่ ข้อความ
     ชิดด้านล่างของจอเหมือนเดิม แต่ตอนนี้ไม่มีปัญหาข้อความ/เนื้อหาโดนบังหรือล้นจอมือถือ
     เพราะ padding/typography ใช้ clamp() แล้ว

3. **`app/layout.tsx` — แก้บั๊ก 2 จุด**
   - เดิม `import type { Metadata }` ถูก import ซ้ำ 2 บรรทัด (บรรทัดสุดท้าย import
     `Metadata, Viewport` อีกที) ซึ่งจริง ๆ แล้วเป็น duplicate identifier ที่ทำให้
     TypeScript error ได้ — รวมเหลือ import เดียว
   - `<Reveal />` ถูก import ไว้ตั้งแต่ต้นแต่ไม่เคยถูกวางใน JSX เลยสักที่ ทำให้
     class `.reveal-ready` ไม่เคยถูกเพิ่มที่ `<html>` และ scroll-reveal effect
     ไม่เคยทำงานจริงตั้งแต่แรก — เพิ่ม `<Reveal />` เข้าไปใน `<body>` แล้ว

4. **หน้า About (`app/about/page.tsx`) — ออกแบบใหม่ทั้งหน้า**
   - ผู้ใช้ขอให้ต่างจากหน้าแรก (ซึ่งเน้นการ์ด/กริด) แต่คงธีมสี/ฟอนต์เดิม
     จึงเปลี่ยนหน้านี้เป็น **โครงไทม์ไลน์แนวตั้ง** ทั้งหน้า:
     - Hero ไม่ใช้วิดีโอ ใช้พื้นเข้มไล่เฉด + เส้นบาง ๆ เกริ่นธีมไทม์ไลน์
     - ส่วน "เรื่องราวของเรา" ที่เดิมเป็น 4 พารากราฟยาวต่อกัน ถูกจัดใหม่เป็น
       4 จุดบนไทม์ไลน์ (จุดเริ่มต้น → จุดเปลี่ยน → ก้าวแรก → ความเชื่อที่ยึดมั่น)
       **เนื้อหา/ข้อความเดิมทุกคำ ไม่มีการเพิ่มหรือตัดข้อมูลใด ๆ** แค่จัดโครงสร้าง
       การนำเสนอใหม่ให้เข้ากับ theme ไทม์ไลน์
     - วิสัยทัศน์/พันธกิจ/คุณค่าองค์กร (PROSPER) ใช้เนื้อหาเดิมทั้งหมด แต่ปรับ
       ให้ต่อเนื่องกับไทม์ไลน์ (อดีต → ทิศทางที่กำลังเดินหน้า)
     - บนจอกว้าง (≥860px) ไทม์ไลน์สลับซ้าย-ขวารอบเส้นกึ่งกลาง, บนมือถือเส้นและ
       จุดชิดซ้าย เนื้อหาทั้งหมดอยู่ขวาเส้นเสมอ (กันเนื้อหาแน่น/อ่านยากบนจอแคบ)
   - แก้ eslint error `react/no-unescaped-entities` (มี `'...'` ตรง ๆ ในข้อความ)
     โดยเปลี่ยนเป็นอัญประกาศเดี่ยวแบบพิมพ์ ‘ ’ ให้เข้าชุดกับเครื่องหมายคำพูดอื่น
     ในหน้าเดียวกันที่ใช้ “ ” อยู่แล้ว
